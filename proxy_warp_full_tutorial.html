3proxy + Cloudflare WARP 透明代理 — 生产级部署教程

一、总体说明（目标）

目标把旁路服务器 (192.168.101.112) 配置为：

  • 使用 Cloudflare WARP 作为系统出口（egress）
  • 提供 3proxy HTTP(8888) / SOCKS5(1080) 服务供临时使用
  • 支持 透明代理（局域网设备无需配置）：将局域网 TCP 流量重定向到本机代理，最后由 WARP 出口发送
  • 生产级别：systemd 管理、开机自启、防火墙持久化、healthcheck、日志、卸载脚本

二、架构图（ASCII）

┌────────┐        ┌────────────────┐       ┌─────────────────┐
│ Client │──HTTP──▶   3Proxy Proxy │──HTTP─▶   WARP Tunnel    │
│        │──HTTPS─▶ (8888 / 1080)  │──TCP──▶ Cloudflare WARP  │
└────────┘        └────────────────┘       └─────────────────┘

数据流：

  1. 客户端将流量发送到 3Proxy (8888 或 1080)
  2. 3Proxy 透明代理统一接管客户端请求
  3. 所有请求通过 Cloudflare WARP 出口发往公网

三、详细安装步骤（手动）

3.1 环境准备

sudo apt update
sudo apt install -y curl wget git nano iptables iproute2 dnsutils     gnupg lsb-release iptables-persistent netfilter-persistent redsocks

3.2 启用 IP 转发

echo "net.ipv4.ip_forward=1" | sudo tee /etc/sysctl.d/99-ipforward.conf
sudo sysctl -p /etc/sysctl.d/99-ipforward.conf

3.3 安装 Cloudflare WARP

curl https://pkg.cloudflareclient.com/pubkey.gpg   | sudo gpg --dearmor -o /usr/share/keyrings/cloudflare-warp.gpg

echo "deb [signed-by=/usr/share/keyrings/cloudflare-warp.gpg] https://pkg.cloudflareclient.com/ $(lsb_release -cs) main"   | sudo tee /etc/apt/sources.list.d/cloudflare-client.list

sudo apt update
sudo apt install -y cloudflare-warp

sudo warp-cli register || true
sudo warp-cli set-mode warp
sudo warp-cli connect
sudo warp-cli status

3.4 安装 3proxy（推荐预编译）

cd /tmp
wget https://github.com/z3APA3A/3proxy/releases/download/0.9.4/3proxy-0.9.4.x86_64.tar.gz -O 3proxy.tar.gz
tar zxvf 3proxy.tar.gz
sudo cp 3proxy/bin/3proxy /usr/local/bin/
sudo chmod +x /usr/local/bin/3proxy
mkdir -p /opt/proxy-system

3.5 安装 redsocks

sudo apt install -y redsocks

base {
 log_debug = off;
 log_info = on;
 log = "file:/var/log/redsocks.log";
 daemon = on;
 redirector = iptables;
}
redsocks {
 local_ip = 127.0.0.1;
 local_port = 12345;
 ip = 127.0.0.1;
 port = 1080;
 type = socks5;
}

五、iptables 透明代理规则（脚本化）

请先确认 WARP 接口名（例如 cloudflare-warp 或 wgcf），并根据需要修改 LAN_CIDR。
/usr/local/bin/warp-transparent.sh
#!/bin/bash
set -e

LAN_CIDR="192.168.101.0/24"
REDSOCKS_PORT=12345
WARP_IF="cloudflare-warp"

iptables -t nat -F warp-prerouting 2>/dev/null || true
iptables -t nat -D PREROUTING -p tcp -j warp-prerouting 2>/dev/null || true
iptables -t nat -X warp-prerouting 2>/dev/null || true

iptables -t nat -N warp-prerouting

iptables -t nat -A warp-prerouting -d 0.0.0.0/8 -j RETURN
iptables -t nat -A warp-prerouting -d 10.0.0.0/8 -j RETURN
iptables -t nat -A warp-prerouting -d 127.0.0.0/8 -j RETURN
iptables -t nat -A warp-prerouting -d 169.254.0.0/16 -j RETURN
iptables -t nat -A warp-prerouting -d 172.16.0.0/12 -j RETURN
iptables -t nat -A warp-prerouting -d 192.168.0.0/16 -j RETURN

iptables -t nat -A warp-prerouting -s $LAN_CIDR -p tcp -j REDIRECT --to-ports $REDSOCKS_PORT
iptables -t nat -A PREROUTING -p tcp -j warp-prerouting

iptables -t nat -A POSTROUTING -o $WARP_IF -j MASQUERADE

netfilter-persistent save

八、一键卸载脚本（生产级）

#!/bin/bash
set -euo pipefail

systemctl stop warp-keepalive warp-transparent 3proxy 2>/dev/null || true
systemctl disable warp-keepalive warp-transparent 3proxy 2>/dev/null || true

rm -f /etc/systemd/system/warp-keepalive.service /etc/systemd/system/warp-transparent.service /etc/systemd/system/3proxy.service
systemctl daemon-reload

pkill -f warp-keepalive -9 2>/dev/null || true
pkill -f redsocks -9 2>/dev/null || true
pkill -f 3proxy -9 2>/dev/null || true

rm -f /usr/local/bin/3proxy /usr/local/bin/warp-transparent.sh /usr/local/bin/warp-keepalive.sh
rm -rf /opt/proxy-system
apt remove -y cloudflare-warp || true
iptables -t nat -F
iptables -F
sysctl -w net.ipv4.ip_forward=0
